# Design Notes

Design-fidelity findings and proposed Figma extensions, owned by **Pixel** (UX Guardian).

Use this folder to record deviations from Figma, missing UI states, responsive-variant
decisions, and reusable-pattern proposals — anything that needs customer review before it
becomes part of the source design.

## Conventions

- One file per topic or feature: `docs/design-notes/<topic>.md`.
- State the Figma frame/link, the gap or proposal, and the status
  (`Open` / `Submitted to customer` / `Approved` / `Rejected`).
- Proposed Figma changes are **subject to customer review and approval** before implementation.
- Link the related PR and ticket.

> This folder is scaffolded for WP1+. It is intentionally empty until features begin.
